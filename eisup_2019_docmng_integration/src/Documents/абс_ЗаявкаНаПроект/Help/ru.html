1.   Общее

Документ Заявка на проект применяется для регистрации нового проекта или для актуализации ранее запущенного проекта.

Каждый документ Заявка на проект проходит по маршруту согласования.

 

2.   Переход к Заявкам на проект

Для того чтобы перейти к списку документов Заявка на проект инициатору необходимо в интерфейсе «Проектный учет» перейти по меню: Документы => Заявка на проект.

 

3.   Заполнение документа Заявка на проект инициатором

Заявку на проект в системе ЕИСУП ТТК оформляет инициатор проекта.

Заявка на проект создается в журнале Заявок на проект типовым образом:

·        Нажатием на кнопку Insert на клавиатуре;

·        Нажатием на кнопку  в командной панели в форме списка документов Заявка на проект.

 

Откроется форма нового документа Заявка на проект следующего вида:

В форме необходимо заполнить все основные реквизиты.

Номер – присваивается автоматически при записи документа;

Дата – дата создания документа. Автоматически заполняется текущей датой.

Статус автоматически при создании нового документа устанавливается в значение Подготовка.

Операция – выбор из списка возможных значений:

·        Создание проекта – для новых проектов;

·        Актуализация данных проекта – для существующих проектов.

По умолчанию при создании нового документа поле «Операция» заполнено значением «Создание проекта».

Группа проектов – указывается, если проект относится к группе проектов.

Организация – указывается организация, в рамках которой будет осуществляться проект.

Проект – указывается базовый проект, если осуществляется актуализация данных проекта:

Вид проекта – выбор из справочника «Виды проектов»:

 

Далее необходимо заполнить Общую информацию по проекту – реквизиты проекта:

·        Наименование – наименование проекта.

·        Код титула – код титула проекта.

·        Дата начала проекта, Дата окончания проекта – сроки проекта. Задаются как период при нажатии на кнопку .

·        Портфель проекта (если есть) – группа проектов с признаком «Case».

·        Отметка доходный (если проект доходный);

·        ТЭО – выбор из справочника «Технико-экономическое обоснование»

·        Заказчик – подразделение, являющееся заказчиком проекта.

·        Руководитель проекта – выбора из справочника «Сотрудники организации» руководителя проекта.

Важно!

Обязательно должны быть заполнены следующие реквизиты:

·         Титул проекта;

·         Наименование;

·         ТЭО (должно быть согласованным);

·         Руководитель проект;

·         Сроки проекта.

 

В поле «Описание проекта» фиксируется дополнительная информация по проекту в произвольной форме.

На закладке «ИТР» отображаются все элементы справочника «ИТР», связанные с выбранным проектом.

Загрузить список ИТР можно заполонить, нажав кнопку «Загрузить ИТР из файла».

 

4.   Особенность заполнения документа Заявка на проект по виду операции Актуализация данных проекта

Для документов Заявка на проект по виду операции «Актуализация данных проекта» необходимо дополнительно заполнить закладки:

·        Доходы;

·        Расходы.

На этих закладках необходимо заполнить накопленными данными по проекту на дату перехода (31.12.2013).

Закладка «Доходы»:

По доходам заполняются две табличные части:

·        Фактические доходы;

·        Входящие платежи.

На вкладке «Фактические доходы» расположена информация по фактическим доходам проекта в разрезе аналитик:

·        Тип контрагента;

·        ЦФУ;

·        ЦФО;

·        Тип сетей/услуг;

·        КВ и ТЗ;

·        Тип расхода;

·        Статья оборотов.

Таблицу «Фактические доходы» можно заполнить данными по проекту, накопленными на дату, указанную в поле «Период данных», нажав кнопку «Заполнить по фактическим данным».

На вкладке «Входящие платежи» расположена информация по фактическим входящим платежам проекта в разрезе аналитик:

·        Контрагент;

·        Договор контрагента;

·        Факт/Опер факт;

·        Оплата/аванс.

Документ (ссылка на платежно-расчетный документ, если есть).

Можно заполнить таблицу «Входящие платежи» данными по проекту, накопленными на дату, указанную в поле «Период данных», нажав кнопку «Заполнить по фактическим данным».

Закладка «Расходы»:

На вкладке «Расходы» отображаются данные по фактическим расходам и исходящим платежам по проекту. Структура данных и порядок работы с ними аналогичны описанным выше (закладка «Доходы»).

Все приведенные таблицы могут быть скорректированы. При проведении заявки на проект в системе, на основании описанных таблиц будут обновлены данные по фактическим доходам и расходам проекта.

 

5.   Согласование Заявки на проект

После того, как все необходимые данные заполнены, документ должен пройти цепочку согласования:

·        Подготовка;

·        Согласование ДТР;

·        Согласование ДЭФ;

·        Разработка ИИТР/ТЭО.

Согласование ТЭО – данный статус доступен только в случае, когда выбранный элемент ТЭО не находится в статусе «Согласован». Переход из этого статуса в следующий невозможен до завершения процесса согласования ТЭО. После того как элемент справочника «ТЭО» будет переведен в статус «Согласован», в такой же статус автоматически будет переведен соответствующий документ «Заявка на проект».

Согласован – этот статус доступен, если в поле «ТЭО» выбран согласованный элемент справочника. После проведения документа в данном статусе, в системе будет создан новый элемент справочника «Проекты» и подставлен в поле «Проект» заявки.

История изменения статусов заявки отображается на закладке «История согласования».
